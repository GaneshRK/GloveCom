Privacy Policy

This Privacy Policy explains how GloveCom (“we”, “our”, “the product”) collects, uses, and protects the data generated by the GloveCom wearable glove, ESP32 processor, TinyML model, and companion mobile application.

1. Data We Collect

GloveCom is designed as a privacy-first assistive device. The following data may be processed:

  • Sensor Data: Finger angle readings generated by the glove sensors.
  • Model Outputs: Recognized gesture labels produced by the TinyML model.
  • App Data: Text or speech output shown within the mobile application.

We do not collect video, audio, or camera data. The system is entirely sensor-based.

2. How Data is Processed

Unlike cloud-based solutions, GloveCom performs gesture recognition using on-device TinyML inference directly on the ESP32 microcontroller.

  • All gesture classification happens offline on the hardware.
  • No raw sensor data is uploaded to external servers.
  • Recognized gestures are sent only to the user's paired mobile device.

3. Data Storage & Retention

By default, GloveCom does not store any personal data on servers or cloud platforms.

  • Temporary sensor data is erased immediately after processing.
  • Text or speech outputs remain only within the user’s mobile app session.
  • No long-term logs, analytics, or history are saved unless explicitly enabled by the user.

4. Mobile App Permissions

Depending on user settings, the mobile app may request:

  • Bluetooth/Wi-Fi: To communicate with the glove.
  • Audio Playback: For text-to-speech output.

No microphone or camera permissions are required.

5. Data Sharing

We do not share, sell, or transfer user data to third parties. All data belongs fully to the user using the device.

6. Your Rights & Control

Users have full control over:

  • Deleting app data or conversation history (if enabled).
  • Managing Bluetooth/Wi-Fi permissions.
  • Opting in for any optional cloud backup features (future scope).

7. Security Practices

GloveCom uses secure Bluetooth/Wi-Fi protocols and ensures all processing is done locally, greatly reducing risk of data exposure.

8. Future Data Features

Some features are planned but not currently active:

  • Cloud-based conversation history (optional)
  • AI sentence prediction
  • Multi-language model updates

Users will be notified before any such feature is introduced.
